20 /*
21  * Defining Generic Expressions
22  *
23  * This module is the place to define custom expressions as you need them for
24  * query execution in the TableScan plan operation. Expressions are basically
25  * a class that provides a match method that is called for every row in the
26  * source tables.
27  *
28  * The two most important macros are:
29  * - DEFINE_EXPRESSION_CLASS( name, field_sequence, logic_sequence)
30  * - REGISTER_EXPRESSION_CLASS( name )
31  *
32  * The first defines and implements the ncessary methods for a given field
33  * definitino. A field definition is a sequnce in terms of C++ macros and has
34  * exactly 4 values. The field name, the type, the comparison operator and the
35  * method name used to extract from json.
36  *
37  * An example for such a sequence is:
38  *
39  * (f1)(hyrise_int_t)(==)(asUInt)
40  *
41  * The parentheses are extremeley important. Now the paremter to the define
42  * macro is bascially a sequence of sequences so it must be defined as
43  *
44  * (seq1...)(seq2...)(seqn..)
45  *
46  * The logical operators are defined in the same way as a sequence of operations:
47  *
48  * ()(&&)(||)
49  *
50  * The first parameter is always empty and used as a placeholder in the code
51  * generation. Please keep the evaluation order and operator precedence in
52  * mind.
53  */
